# Pull Request Template

## `.github/pull_request_template.md`

```markdown
<!--
PR 제목은 다음과 같은 형식으로 작성해주세요.
- [Feat]: 새로운 기능 추가
- [Fix]: 버그 수정
- [Docs]: 문서 수정
- [Design]: UI/뷰 관련 수정
- [Refactor]: 코드 리팩토링
- [Test]: 테스트 코드 추가/수정
- [Chore]: 빌드 관련 파일 수정, 설정 등
-->

## 작업 요약

<!-- 이 PR의 핵심 목표를 한두 문장으로 요약해주세요. -->
<!-- 예: 앱 실행 시 다음 버스로 자동 스크롤하는 기능을 구현했습니다. -->

## 상세 작업 내용

<!-- 이 PR에서 수행한 구체적인 작업들을 항목별로 나열해주세요. -->

- [x] **항목1**
- [x] **항목2**
- [x] **항목3**

## 스크린샷

<!--
스크린샷 필요 여부:
- 필수: UI 변경, 컴포넌트 추가/수정, UI 버그 수정
- 생략 가능: 로직 변경, 설정/문서 변경

UI 변경이 없다면 이 섹션을 삭제하거나 "N/A"로 표시해주세요.
-->

|          변경 전           |          변경 후           |
| :------------------------: | :------------------------: |
| <!-- 이미지나 GIF 첨부 --> | <!-- 이미지나 GIF 첨부 --> |

## 테스트

<!-- 테스트 완료 여부를 체크해주세요 -->

- [ ] 유닛 테스트 통과
- [ ] 시뮬레이터 테스트 완료
- [ ] 실기기 테스트 완료 (해당되는 경우)

## 이슈 및 특이사항

<!--
작업 중 발생한 이슈, 해결 과정, 고민했던 점이나 다음에 참고할 만한 특이사항을 기록해주세요.
미래의 나 자신을 위한 메모라고 생각하고 편하게 작성하면 좋습니다.
-->

- **이슈**:
- **특이사항**:

## 다음에 할 일

<!-- 이 작업 이후에 진행할 다음 작업 계획을 간략하게 작성해주세요. -->

-
-

## 관련 이슈

<!-- 관련 이슈가 있다면 연결해주세요 -->
<!-- Close #이슈번호 형식으로 작성하면 PR 머지 시 이슈가 자동으로 닫힙니다 -->

Close #
```

## PR 제목 예시

| 타입         | 예시                                       |
| ------------ | ------------------------------------------ |
| `[Feat]`     | `[Feat]: 다음 버스 자동 스크롤 기능 추가`  |
| `[Fix]`      | `[Fix]: 공휴일 시간표 표시 버그 수정`      |
| `[Design]`   | `[Design]: 다음 버스 셀 강조 스타일 적용`  |
| `[Refactor]` | `[Refactor]: DateService 시간 계산 분리`   |
| `[Test]`     | `[Test]: findNextBus 유닛 테스트 추가`     |
| `[Docs]`     | `[Docs]: README 설치 방법 업데이트`        |
| `[Chore]`    | `[Chore]: Xcode 프로젝트 설정 정리`        |

## PR 본문 작성 팁

### 작업 요약

```markdown
## 작업 요약

앱 실행 시 현재 시간 이후 가장 가까운 버스로 자동 스크롤하는 기능을 구현했습니다.
막차 이후에는 "오늘 운행이 종료되었습니다" 메시지를 표시합니다.
```

### 상세 작업 내용

```markdown
## 상세 작업 내용

- [x] **DateService에 findNextBus 메서드 추가**
- [x] **MainView에 자동 스크롤 로직 적용**
- [x] **NextBusCell 강조 스타일 구현**
- [x] **운행 종료 상태 처리**
- [x] **유닛 테스트 추가**
```

### 스크린샷

```markdown
## 스크린샷

|          변경 전           |          변경 후           |
| :------------------------: | :------------------------: |
| ![before](url)             | ![after](url)              |
```

## 체크리스트

PR 생성 전 확인사항:

- [ ] PR 제목이 컨벤션을 따르는가? (`[Type]: 설명`)
- [ ] 작업 요약이 명확한가?
- [ ] 상세 작업 내용이 체크리스트로 작성되었는가?
- [ ] UI 변경 시 스크린샷이 첨부되었는가?
- [ ] 테스트를 완료했는가?
- [ ] 관련 이슈가 연결되었는가?
