# TDD Status

## Context

- Current plan: @plan.md

## Task

`plan.md` 파일을 분석하여 현재 TDD 진행 상황을 보고하세요.

## 분석 항목

1. **완료된 테스트**: `- [x]` 항목 수
2. **남은 테스트**: `- [ ]` 항목 수
3. **진행률**: (완료 / 전체) * 100%
4. **다음 테스트**: 첫 번째 미완료 테스트

## 출력 형식

```
📊 TDD 진행 상황
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 완료: X개
⏳ 남음: Y개
📈 진행률: Z% [████████░░░░░░░░░░░░]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 다음 테스트:
   testMethodName
   설명: [테스트 설명]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📝 완료된 테스트 목록:
   ✓ testMethod1
   ✓ testMethod2
   ...
```

plan.md 파일이 없으면 `/tdd-plan`으로 먼저 계획을 수립하라고 안내하세요.
